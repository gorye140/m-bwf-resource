# m-bwf-resource

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以资源文件的形式存放，不针对任何特定域名或网站，仅作为静态内容存储与展示的集合。

## 项目简介

m-bwf-resource 是一个轻量级的静态资源仓库，主要存放各类独立 HTML 页面。每个页面均为自包含的单一文件，不依赖外部服务器或后端逻辑，适合作为历史存档、示例展示或实验性内容的载体。

## 目录说明

```
m-bwf-resource/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 页面引用的公共资源（CSS、JS、图片等）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件为一个独立页面，命名建议反映内容或用途。
- `assets/`：可选目录，用于存放页面间共享的样式、脚本或媒体文件。

## 页面归档说明

- 所有页面均为独立 HTML 文件，可直接在浏览器中打开运行。
- 页面内容不包含对任何外部服务或域名的硬编码依赖。
- 归档遵循“一次提交，只增不改”的原则，历史版本可通过 Git 提交记录追溯。
- 不建议在页面中嵌入跟踪、统计或外部请求逻辑，以保持归档的纯粹性。

## 维护说明

- 欢迎提交新页面：将 HTML 文件放入 `pages/` 目录，并附上简要说明。
- 更新已有页面时，请尽量保留原文件，新增版本可另存为新文件。
- 仓库不承担页面内容的审核责任，但建议提交内容清晰、合法且不包含敏感信息。
- 如有疑问或建议，可通过 Issue 讨论。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/m-bwf-resource.git
   ```
2. 直接打开 `pages/` 下的 HTML 文件即可查看。
3. 也可通过 GitHub Pages 或其他静态托管服务部署（需自行配置）。

---

本仓库仅作为静态 HTML 页面的存档与分享用途，不提供任何形式的服务或内容承诺。
